# Git 워크플로우 자동화

이 워크플로우는 사용자의 Git 작업을 도와줍니다.
프로젝트 경로: `c:\Users\kimdy\Desktop\Infam chatbot 2.0`

## 작업 유형별 실행 방법

사용자가 `/git-workflow`를 호출하면, 먼저 어떤 작업을 원하는지 물어보세요.
아래 작업 중 해당하는 것을 실행합니다.

---

### 1. 현재 상태 확인

// turbo
```
git status
```

// turbo
```
git log --oneline -5
```

결과를 한글로 요약하여 사용자에게 보고합니다:
- 수정된 파일 수
- 스테이징된 파일 수
- 커밋되지 않은 변경사항 요약
- 최근 5개 커밋 요약

---

### 2. 변경사항 스테이징 & 커밋

// turbo
```
git status
```

// turbo
```
git diff --stat
```

변경사항을 분석한 뒤:
1. 스테이징할 파일 목록을 사용자에게 보여줍니다
2. 사용자 확인 후 `git add` 실행
3. 변경 내용을 분석하여 한글 커밋 메시지 제안
4. 사용자 확인 후 `git commit` 실행

---

### 3. 브랜치 관리

// turbo
```
git branch -a
```

사용자에게 현재 브랜치 상태를 보여준 뒤:
- **새 브랜치 생성**: `git checkout -b <브랜치명>`
- **브랜치 전환**: `git checkout <브랜치명>`
- **브랜치 병합**: `git merge <브랜치명>`
- **브랜치 삭제**: `git branch -d <브랜치명>`

사용자에게 원하는 작업을 물어보고 실행합니다.

---

### 4. 원격 저장소 동기화

// turbo
```
git remote -v
```

// turbo
```
git status
```

상태를 확인한 뒤:
- **푸시**: `git push origin <현재브랜치>`
- **풀**: `git pull origin <현재브랜치>`
- **원격 추가**: `git remote add origin <URL>`

사용자에게 원하는 작업을 물어보고 실행합니다.

---

### 5. 히스토리 조회

// turbo
```
git log --oneline --graph --all -20
```

결과를 한글로 읽기 쉽게 요약하여 보여줍니다:
- 각 커밋의 날짜, 작성자, 메시지
- 브랜치 구조를 시각적으로 설명

---

## 주의사항

- 모든 Git 명령어는 프로젝트 루트 디렉터리에서 실행하세요
- `git push`나 `git reset --hard` 같은 위험한 명령은 반드시 사용자 확인 후 실행
- 오류 발생 시 한글로 원인과 해결 방법을 안내하세요
