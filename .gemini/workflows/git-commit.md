# Git 커밋 메시지 자동 생성

이 워크플로우는 현재 변경사항을 분석하여 한글 커밋 메시지를 자동으로 생성합니다.
프로젝트 경로: `c:\Users\kimdy\Desktop\Infam chatbot 2.0`

## 실행 단계

### 1단계: 변경사항 확인

// turbo
```
git status
```

// turbo
```
git diff --staged
```

스테이징된 변경사항이 없으면:

// turbo
```
git diff
```

사용자에게 변경사항 요약을 한글로 보여줍니다.

---

### 2단계: 스테이징되지 않은 파일이 있다면

사용자에게 어떤 파일을 스테이징할지 물어봅니다:
- 전체 스테이징: `git add .`
- 선택 스테이징: `git add <파일1> <파일2> ...`

사용자 확인 후 스테이징을 실행합니다.

---

### 3단계: 커밋 메시지 생성

`git diff --staged` 결과를 분석하여 **Conventional Commits** 형식의 한글 커밋 메시지를 생성합니다.

#### 커밋 타입 가이드

| 타입 | 사용 시점 | 예시 |
|------|----------|------|
| `feat` | 새로운 기능 추가 | `feat: 챗봇 응답 복사 기능 추가` |
| `fix` | 버그 수정 | `fix: 채팅 스크롤 오류 수정` |
| `docs` | 문서 수정 | `docs: README 설치 방법 업데이트` |
| `style` | 코드 포맷팅 (기능 변경 없음) | `style: CSS 들여쓰기 정리` |
| `refactor` | 코드 리팩토링 | `refactor: RAG 검색 로직 개선` |
| `chore` | 빌드/설정 변경 | `chore: .gitignore 업데이트` |
| `perf` | 성능 개선 | `perf: 이미지 로딩 최적화` |
| `test` | 테스트 추가/수정 | `test: FAQ 응답 테스트 추가` |

#### 메시지 형식
```
<타입>: <한글 요약> (50자 이내)

<빈 줄>
<상세 설명 (선택사항, 한글)>
- 변경사항 1
- 변경사항 2
```

#### 생성 규칙
1. `git diff --staged` 내용을 꼼꼼히 분석합니다
2. 변경 파일의 종류와 수정 내용을 파악합니다
3. 적절한 커밋 타입을 선택합니다
4. 변경의 **핵심 의도**를 50자 이내 한글로 요약합니다
5. 여러 파일이 변경된 경우 상세 설명에 주요 변경사항을 나열합니다

---

### 4단계: 사용자 확인

생성된 커밋 메시지를 사용자에게 보여주고 확인을 요청합니다:
- ✅ 승인 → 커밋 실행
- ✏️ 수정 요청 → 메시지 수정 후 다시 확인
- ❌ 취소 → 워크플로우 종료

---

### 5단계: 커밋 실행

사용자가 승인한 커밋 메시지로 커밋합니다:

```
git commit -m "<승인된 메시지>"
```

커밋 완료 후 결과를 한글로 보고합니다:
- 커밋 해시
- 변경된 파일 수
- 추가/삭제된 줄 수

---

### 6단계 (선택): 푸시 여부

커밋 후 원격 저장소에 푸시할지 물어봅니다:
- 예 → `git push origin <현재브랜치>` 실행
- 아니오 → 종료

---

